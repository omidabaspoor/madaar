# گزارش بازطراحی و بازنویسی صفحه‌ی جلسه‌ی آنلاین مَدار

> تاریخ: ۱۴۰۳/۰۴/۰۸ (۲۰۲۶-۰۶-۲۸)  
> وضعیت: ✅ بازطراحی کامل + صفحات مدیریت/لیست + API امن

---

## آنچه انجام شد

با توجه به درخواست شما برای **بازطراحی کامل صفحه‌ی جلسه‌ی آنلاین** و **فعلاً بی‌خیال سیستم اتصال (WebRTC/P2P/Jitsi)**:

### ۱. اتاق جلسه (`online_room.php`)
- صفحه‌ی قبلی به‌طور کامل بازطراحی شد.
- فوکوس اصلی: **تخته‌ی تعاملی** + **چت زنده** + **لیست شرکت‌کنندگان** + **مدیریت دسترسی‌ها**.
- اتصال تصویری/صوتی (P2P/Jitsi) حذف شد؛ فقط UI و سیستم permissions باقی مانده است.
- طراحی responsive برای دسکتاپ و موبایل.
- صفحه‌ی خطا با اطلاعات دقیق و لینک بازگشت.

### ۲. تخته‌ی تعاملی (Whiteboard)
- سیستم قدیم snapshot تصویر به **vector strokes** تغییر کرد.
- ابزارها: قلم، خط، مستطیل، دایره، متن، پاک‌کن.
- رنگ‌ها و ضخامت قابل تنظیم.
- آپلود PDF به‌عنوان پس‌زمینه (فقط میزبان).
- دانلود تخته به‌صورت PNG.
- پاک‌کردن همه (فقط میزبان).
- همگام‌سازی زنده‌ی strokes با polling.
- نمایش وضعیت همگام‌سازی (فعال / در حال ذخیره / خطا).

### ۳. چت زنده (Chat)
- چت real-time با polling.
- پیام‌های خود (right) و دیگران (left).
- پیام‌های سیستمی.
- تعداد پیام‌های خوانده‌نشده روی دکمه‌ی چت.
- غیرفعال‌سازی خودکار چت وقتی جلسه تمام می‌شود یا دسترسی بسته است.

### ۴. لیست شرکت‌کنندگان (Participants)
- نمایش میزبان و دانش‌آموزان.
- بروزرسانی وضعیت آنلاین/آفلاین با polling.
- میزبان می‌تواند دسترسی‌ها را مدیریت کند.

### ۵. مدیریت دسترسی‌ها (Permissions)
- میزبان می‌تواند از تنظیمات سمت راست: میکروفون، دوربین، اشتراک صفحه، تخته، چت را روشن/خاموش کند.
- دانش‌آموزان می‌توانند درخواست دسترسی بدهند.
- میزبان درخواست‌ها را می‌بیند و تأیید/رد می‌کند (با badge روی تب تنظیمات).
- تغییرات دسترسی به‌صورت زنده برای همه اعمال می‌شود.

### ۶. دست بلند کردن و واکنش‌ها
- دانش‌آموز می‌تواند دست بلند کند؛ میزبان نام او را در بنر می‌بیند.
- واکنش‌های شناور (👏 ❤️ 👍 🔥 ⭐ 😂 😮).

### ۷. تایمر و نشانگر LIVE
- تایmer زنده از زمان شروع جلسه.
- نشانگر LIVE با پالس.
- تعداد شرکت‌کنندگان.
- شروع خودکار جلسه هنگام ورود میزبان (اگر scheduled باشد).
- پایان جلسه با دکمه‌ی «پایان» (میزبان) یا «خروج» (دانش‌آموز).

### ۸. API (`api/online_room.php`)
- کاملاً بازنویسی و امن‌سازی شد.
- حذف **soft mode CSRF** (که قبلاً CSRF را دور می‌زد).
- CSRF برای همه‌ی عملیات غیر GET الزامی است.
- endpointهای جدید برای strokes:
  - `whiteboard_stroke_save`
  - `whiteboard_stroke_list`
  - `whiteboard_stroke_clear`
- endpointهای legacy snapshot حفظ شدند.
- اعتبارسنجی نقش و دسترسی در هر endpoint.

### ۹. صفحات مدیریت و لیست
- **`admin/online_sessions.php`**: ایجاد، ویرایش، حذف، شروع، پایان جلسات + انتخاب دانش‌آموزان + دسترسی‌ها.
- **`student/online_sessions.php`**: لیست جلسات دعوت‌شده + دکمه‌ی ورود به اتاق.

### ۱۰. دیتابیس
- جدول جدید `whiteboard_strokes` برای ذخیره‌ی strokes.
- Migration به‌روزرسانی شد.
- `install.php` شامل `upgrade_online_sessions.sql` شد.

## نکات مهم

- **اتصال تصویری/صوتی (P2P/Jitsi) فعلاً حذف شد**؛ دکمه‌های میکروفون/دوربین/اشتراک صفحه در UI وجود دارند اما فقط permissions را مدیریت می‌کنند.
- **تخته به‌صورت vector strokes همگام می‌شود**؛ نه snapshot تصویر. این بسیار سبک‌تر و collaborative‌تر است.
- **همه‌ی actionها با CSRF محافظت شده‌اند** و soft mode حذف شده است.
- **همه‌ی endpointها دسترسی را بررسی می‌کنند** (host/advisor/student).
- **Polling** برای زنده‌نگه‌داشتن چت، تخته، دسترسی‌ها، دست و واکنش‌ها استفاده می‌شود.

## تست پیشنهادی

### تست میزبان:
1. لاگین مشاور → منوی «جلسات آنلاین».
2. ایجاد جلسه با انتخاب چند دانش‌آموز و تنظیم دسترسی‌ها.
3. کلیک «شروع و ورود» → ورود به اتاق.
4. رسم روی تخته، تایپ متن، آپلود PDF، پاک کردن.
5. ارسال پیام در چت.
6. تغییر دسترسی‌ها از تب «تنظیمات».
7. پایان جلسه.

### تست دانش‌آموز:
1. لاگین دانش‌آموز → منوی «جلسات آنلاین».
2. کلیک «ورود به اتاق».
3. مشاهده‌ی تخته و چت.
4. ارسال پیام در چت (اگر فعال باشد).
5. بلند کردن دست و واکنش.
6. درخواست دسترسی به تخته (اگر بسته باشد).
