- حفظ ظاهر و منطق تخته‌ی فعلیِ فایل onlineclass_raw.html
- یکپارچه‌سازی با هویت بصری، نقش‌ها، نشست‌ها، مجوزها و داده‌های مَدار

تصمیم‌های ادغام:
1) UI/UX اتاق جلسه
- بدنه‌ی HTML/CSS/JS اتاق را بر اساس طراحی خام کاربر بازطراحی می‌کنیم
- متن‌های فنی حذف می‌شوند
- لوگو، نام جلسه، وضعیت، کنترل‌ها و نقش‌ها با داده‌های واقعی مَدار پر می‌شوند

2) تخته مجازی
- ظاهر و رفتار toolbar/canvas/pdf/selection دقیقا از طرح خام گرفته می‌شود
- ذخیره/لود به API فعلی whiteboard_save / whiteboard_load متصل می‌شود
- اگر لازم باشد snapshot به‌صورت image/webp + state json ذخیره می‌شود

3) ویدئو/صدا/اشتراک صفحه
- لایه‌ی P2P فعلی حفظ و با UI جدید سینک می‌شود
- کلیدهای mic/cam/screen/hand/chat/people/board به state واقعی وصل می‌شوند
- برای نقش دانش‌آموز، درخواست دسترسی‌ها به سیستم مجوز فعلی وصل می‌شود

4) پنل‌ها
- admin/online_sessions.php: صفحه مدیریت کامل، کارت‌ها، ایجاد جلسه، وضعیت‌ها، لیست کلاس‌ها، عملیات سریع
- student/online_sessions.php: صفحه جلسات من با وضعیت جلسه، زمان، دکمه ورود و تجربه بهتر

5) عدم نمایش متن فنی
- در UI عمومی هیچ اشاره‌ای به رایگان/هیبرید/Jitsi/WebRTC/P2P نمی‌شود
- فقط تجربه‌ی کاربری تمیز و محصول‌محور نمایش داده می‌شود
