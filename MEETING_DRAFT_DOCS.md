## 🎯 فلوی جدید (۴ مرحله)

```
۱. 📝 پر کردن فرم
   ├── نوع جلسه: مشاوره / کلاس
   ├── دانش‌آموز(ها): انتخاب از لیست
   ├── تاریخ: ✓ اجباری
   ├── ساعت: ✓ اجباری (جدید!)
   ├── عنوان: اختیاری
   └── توضیحات: اختیاری

۲. 💾 ذخیره پیش‌نویس
   ├── وضعیت: draft
   ├── هیچ پیامکی ارسال نمی‌شود
   └── دانش‌آموز چیزی نمی‌بیند

۳. ✏️ ویرایش یا حذف پیش‌نویس
   ├── ویرایش: تغییر اطلاعات → ذخیره مجدد
   └── حذف: پاک شدن کامل

۴. ✅ تأیید نهایی + ارسال پیامک
   ├── وضعیت: scheduled
   ├── پیامک به همه‌ی دانش‌آموزان
   ├── اعلان درون‌برنامه‌ای
   └── قابل لغو
```

## 🔄 تغییرات دیتابیس

### Migration (`upgrade_meeting_draft.sql`):
```sql
ALTER TABLE consultation_sessions
MODIFY COLUMN status ENUM('draft','scheduled','completed','cancelled') 
NOT NULL DEFAULT 'draft';

ALTER TABLE consultation_sessions
ADD COLUMN IF NOT EXISTS draft_group_id VARCHAR(40) DEFAULT NULL AFTER session_type;

UPDATE consultation_sessions SET status='scheduled' WHERE status='draft';
```

### جدول وضعیت‌ها:
```
draft       → پیش‌نویس (SMS ارسال نشده)
scheduled   → تأیید شده (SMS ارسال شده)
completed   → برگزار شده
cancelled   → لغو شده
```

## 🛠 توابع جدید (meetings.php)

```php
// ذخیره‌ی پیش‌نویس (بدون SMS)
meetings_save_draft($advisorId, $studentIds, $title, $date, $time, $notes, $type, $existingDraftGroupId = null)

// دریافت پیش‌نویس‌های یک مشاور (گروه‌بندی شده)
meetings_drafts_for_advisor($advisorId)

// دریافت اطلاعات یک گروه پیش‌نویس برای ویرایش
meetings_get_draft_for_edit($advisorId, $draftGroupId)

// حذف یک گروه پیش‌نویس
meetings_delete_draft($advisorId, $draftGroupId)

// تأیید نهایی + ارسال SMS به همه
meetings_confirm($advisorId, $draftGroupId)
```

## 🧪 سناریوهای تست

### سناریو ۱: مشاور تازه‌کار
```
1. فرم رو پر می‌کنه
2. ساعت رو یادش می‌ره → ارور: "ساعت شروع الزامی است"
3. دانش‌آموز رو انتخاب نمی‌کنه → ارور
4. ذخیره پیش‌نویس → کارت پیش‌نویس ظاهر می‌شه
5. ویرایش → ساعت رو اضافه می‌کنه
6. تأیید نهایی → SMS ارسال
```

### سناریو ۲: کلاس گروهی
```
1. نوع: کلاس
2. ۱۰ دانش‌آموز انتخاب (۸ موبایل، ۲ بدون)
3. شمارنده: "۸ پیامک، ۲ بدون موبایل"
4. تأیید → هشدار: "۲ دانش‌آموز موبایل ندارند"
5. تأیید نهایی → ۸ SMS ارسال، ۲ بدون
```

### سناریو ۳: تغییر بعد از ذخیره
```
1. پیش‌نویس ذخیره شد
2. مشاور متوجه میشه یکی از دانش‌آموزان نباید باشه
3. ویرایش → حذف اون دانش‌آموز از لیست
4. ذخیره مجدد → فقط همون یکی حذف شد
5. تأیید نهایی → SMS به بقیه
```
